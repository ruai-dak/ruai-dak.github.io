The first module deals with mainly the basics of github, setting it up and making a website to document the work we are doing.They are covered in depth below

Introduction to github

GitHub is a web-based platform that uses Git, a version control system, to help developers manage and collaborate on software development projects. It provides a central place to store code, track changes, and work together on projects, facilitating collaboration and version control for software development.

Since i had a functional github account already i used that to document what i will be doing over the course of the next few weeks

Below is a showing of how the github dashboard looks like

The GitHub dashboard is your main hub after logging in. It helps you track updates, manage repositories, and collaborate with others efficiently.

Top Navigation

At the top of every page, you will find:

  • Search bar whre you will Find repos, users, or code.
  • + Create new projects or gists.
  • Bell icon View notifications.
  • Profile menu Access your profile, settings, or sign out.

Dashboard Sections

  • Feed - Shows activity from people and repos you follow.
  • Sidebar - Quick links to your repositories and organizations.
  • Explore - Suggestions for trending projects and topics.

Inside a Repository

When you open a repo, you’ll see tabs like:

  • Code - Browse files
  • Issues - Track bugs or tasks
  • Pull Requests - Review code changes
  • Actions - Automate tasks

Your Profile

Your GitHub profile shows your bio, repositories, activity graph, and pinned projects.

Git

Git is a distributed version control system that tracks versions of files. It is often used to control source code by programmers who are developing software collaboratively.

I got to familiarize myself with these git commands

1.git status --this is used to check if a folder is a git repo
2.git clone --this is used to download the folder from github to your computer
3.git add --this is used to specify what files to add to youre Repository
4.git commit --this is used to specify the changes you have made
5.git push --this pushes the code on your local computer to your github repository
